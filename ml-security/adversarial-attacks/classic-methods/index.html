    对抗攻击方法演进：从 FGSM 到 C&W¶

    四篇经典论文，一条演化主线 —— 为什么一个 ε = 0.007 的扰动能让模型把熊猫认成长臂猿？攻击方法又是怎么从「一步到位」进化到「迭代优化」的？

    0. 先统一语言：攻击和防御在干什么¶

    在进入具体方法之前，把数学框架说清楚。

    攻击：给定原始样本 x 和标签 y，在扰动约束集合 S 内，找一个扰动 δ，让损失函数最大化：

    \[ \max_{\delta \in S} L(\theta, x + \delta, y) \]

    约束 S 通常是一个范数球（比如 ‖δ‖∞ ≤ ε），保证扰动「人眼看不出」。

    防御：在这个攻击设定下，训练一个鲁棒模型，让最坏情况下的期望损失最小：

    \[ \min_\theta \rho(\theta),\quad \rho(\theta) = \mathbb{E}_{(x,y) \sim D}\left[\max_{\delta \in S} L(\theta, x + \delta, y)\right] \]

    这就是 min-max 框架——Madry 等人后来把它形式化成对抗机器学习的标准范式。理解这个框架，才能看懂后面四种方法各自的定位。

    负责什么 对应方法
    内层 max 攻击：找到最强的对抗样本 FGSM → BIM → PGD → C&W
    外层 min 防御：训练鲁棒模型 对抗训练（用内层攻击生成样本）

    1. FGSM：为什么一步就够了？¶

    论文：Goodfellow et al., Explaining and Harnessing Adversarial Examples (ICLR 2015)

    1.1 核心洞察：不是非线性，是线性¶

    2015 年之前，大家普遍认为对抗样本的存在是因为神经网络的高度非线性——函数太复杂、太弯曲，所以在数据点附近出现了意想不到的陡峭区域。

    Goodfellow 说：不需要非线性，高维空间里的线性行为就足够产生对抗样本。

    直觉是这样的：

    • 输入 x 是一个高维向量（比如 224×224×3 ≈ 150,000 维）
    • 扰动 η 的每个分量都很小（比如 ‖η‖∞ < ε）
    • 但 150,000 个「很小」加起来就是一个不容忽视的量

    具体来说，考虑一个简单的线性模型 f(x) = w^T x + b。加上扰动后：

    \[ w^T(x + η) = w^T x + w^T η \]

    扰动带来的变化量是 w^T η。如果取 η = ε · sign(w)——即扰动方向和权重方向一致——那么：

    \[ w^T η = ε · \|w\|_1 \]

    而 ‖w‖₁ 在高维情况下可以很大（因为它是所有权重绝对值之和，150,000 维累加起来）。所以即使 ε 很小，对输出的影响也可能非常大。

    1.2 FGSM 公式¶

    把上面的直觉推广到非线性模型，扰动方向不再用权重 w，而是用损失函数对输入的梯度：

    \[ \boxed{x' = x + ε \cdot \text{sign}\left(\nabla_x J(\theta, x, y)\right)} \]

    逐项解释：

    • ∇_x J(θ, x, y)：损失对输入的梯度——告诉你「往哪个方向动像素，loss 涨最快」
    • sign(·)：只取符号（+1 或 −1），不关心梯度大小。这保证了每个像素改动量一致，都是 ±ε
    • ε：扰动强度，控制「攻击力度」

    为什么用符号而不是直接用梯度值？ 两个原因：

    1. 均匀性：每个像素改动量都是 ±ε，扰动分布均匀，不易被检测
    2. 速度：一步到位，不需要迭代

    这就是 "Fast Gradient Sign Method" 名字的由来——快，且只用梯度的符号。

    1.3 在 Logistic Regression 上的推导（可选）¶

    很多人对 FGSM 和逻辑回归的关系没搞清。这里给出完整推导，帮助理解线性假设。

    设定：二分类，标签 y ∈ {−1, 1}，预测 P(y=1) = σ(w^T x + b)，损失用 softplus：τ(z) = log(1 + e^z)。

    逻辑回归的损失函数：

    \[ J(w, x, y) = τ\left(-y(w^T x + b)\right) \]

    对 x 求梯度（链式法则，softplus 的导数就是 sigmoid）：

    \[ \nabla_x J = τ'\left(-y(w^T x + b)\right) · (-y) · w = \frac{e^{-y(w^T x + b)}}{1 + e^{-y(w^T x + b)}} · (-y) · w \]

    代入 FGSM 公式：

    \[ η = ε · \text{sign}(\nabla_x J) = ε · \text{sign}\left(\frac{e^{-y(w^T x + b)}}{1 + e^{-y(w^T x + b)}} · (-y) · w\right) \]

    关键观察：sigmoid 函数的值在 (0, 1) 之间，前面的系数恒正。所以符号完全由 -y · w 决定：

    • y = 1：η = ε · \text{sign}(-w) = -ε · \text{sign}(w)
    • y = −1：η = ε · \text{sign}(w) = -y · ε · \text{sign}(w)

    统一写成：

    \[ \boxed{η = -y · ε · \text{sign}(w)} \]

    这个结果的直觉：

    • 因为 sign(w) 就是权重 w 的符号方向
    • 所以 w^T · sign(w) = \|w\|₁
    • 扰动带来的输出变化量 w^T η = -y · ε · \|w\|₁

    注意 \|w\|₁ 在高维数据（比如图像）上可以非常大！即使 ε 很小（比如 0.007），ε · ‖w‖₁ 也足以让样本跨过决策边界。

    这就是线性假设的核心：对抗样本不是「模型太复杂」造成的，而是「高维空间里 L₁ 范数太大」造成的。只要维度够高，一个微小扰动就能累积出巨大影响。

    2. BIM：从一跳到多步¶

    论文：Kurakin et al., Adversarial Machine Learning at Scale (ICLR 2017)

    2.1 FGSM 不够强¶

    Goodfellow 的 FGSM 在 MNIST 上很有效，但在 ImageNet 上成功率只有 63% — 69%。原因很简单：对大图来说，一步跨出去不一定能跨过决策边界。

    Kurakin 等人的改进思路很直接：多次迭代。

    2.2 BIM 公式¶

    每一步走一小步（步长 α，通常远小于 ε），走完 N 步后总扰动用 ε 裁减：

    \[ x_0 = x \]
    \[ x_{k+1} = \text{Clip}_{x, ε}\left(x_k - α · \text{sign}\left(\nabla_x J(x_k, y_{\text{target}})\right)\right) \]

    和 FGSM 有两个关键区别：

    FGSM BIM
    步数 1 步 多步迭代（如 10-40 步）
    攻击目标 无目标（远离正确类就行） 有目标（推向特定错误类）

    注意 BIM 公式里是 减号而不是加号——因为攻击目标从「最大化正确类的 loss」变成了「最小化目标类的 loss」。目标类别通常选模型最不可能是的那个类（即模型置信度最低的类）。

    2.3 BIM 也被称为 Iterative FGSM (I-FGSM)¶

    BIM 本质上就是 FGSM 的迭代版本，所以也常被称为 I-FGSM（Iterative FGSM）。

    2.4 批量对抗训练¶

    Kurakin 等人还提出了一个实用的对抗训练方案：在训练时混入对抗样本，用可控的比例：

    \[ \tilde{J}(\theta, x, y) = \alpha · J(\theta, x, y) + (1 - \alpha) · J(\theta, x^{\text{adv}}, y) \]

    其中 α 控制干净样本和对抗样本的比例。这让对抗训练可以在大规模数据集上高效进行。

    实验中一个有趣的发现：FGSM 的可迁移性远强于 BIM。BIM 生成的对抗样本太「定制化」了——它对当前模型有效，但换个模型就不灵了。FGSM 因为「粗糙」，反而更容易迁移。这对黑盒攻击有启发意义。

    3. PGD：目前为止最强的 first-order 攻击¶

    论文：Madry et al., Towards Deep Learning Models Resistant to Adversarial Attacks (ICLR 2018)

    3.1 Min-Max 框架的形式化¶

    Madry 等人做了两件重要的事，一个是形式化，一个是方法改进。

    形式化：把对抗攻击和防御统一成 min-max 框架（就是本文第 0 节那个公式）。这个框架的影响远超这篇论文本身，后来几乎所有的对抗攻防研究都默认采用这个范式。

    3.2 PGD 的改进：随机初始化¶

    PGD 和 BIM 的本质区别就一个：初始点在随机位置，而不是原图。

    \[ x_0 = x + U(-\epsilon, \epsilon) \]

    然后在范数球内迭代：

    \[ x_{k+1} = \Pi_{B(x, \varepsilon)}\left(x_k + \alpha · \text{sign}\left(\nabla_x J(x_k, y)\right)\right) \]

    其中 Π_{B(x, ε)} 表示投影回 x 的 ε-范数球内。

    为什么要随机初始化？

    BIM 总是从同一个位置（原图）出发，它可能陷入一个局部最优——找到了一个「看起来像是」对抗样本的点，但实际上还有更强的攻击隐藏在附近的某个位置。

    随机初始化 = 每次从不同的地方开始搜索 = 更有可能找到全局最优点。这就好比你想在一张地图上找最低点：从同一个地点出发，你可能每次都走到同一个山谷；从不同地点出发，你可能发现一个更深的盆地。

    3.3 PGD vs BIM 的区别总结¶

    BIM (I-FGSM) PGD
    起点 原图 x 随机扰动后的 x + U(-ε, ε)
    迭代 固定步数 更多迭代（如 40 步）
    搜索策略 确定性 随机 + 多步
    攻击强度 强 更强
    论文 Kurakin et al. (2017) Madry et al. (2018)

    一句话：PGD = BIM + 随机初始化 + 更多迭代轮数。

    3.4 实验洞察：模型容量越大越鲁棒¶

    Madry 等人做了一个重要的实验观察：随着模型容量（宽度）增大，不仅在干净数据上表现更好，对抗训练后的鲁棒性也在持续提升。这个趋势和对干净数据不同——干净数据上模型大到一定程度后收益递减，但对抗鲁棒性似乎是「越大越好」。

    这暗示了一个可能性：鲁棒性需要更多的模型容量来「记住」更复杂的决策边界。

    4. C&W：换一种思路——直接优化扰动¶

    论文：Carlini & Wagner, Towards Evaluating the Robustness of Neural Networks (IEEE S&P 2017)

    4.1 告别梯度符号¶

    前面三种方法（FGSM、BIM、PGD）都是梯度方向的变种。C&W 说：我们直接求解一个优化问题。

    思路朴素得令人意外：我们想要最小化扰动 δ，同时保证 x + δ 被错分。翻译成数学：

    \[ \min_{\delta} \|\delta\|_p \quad \text{s.t.} \quad C(x + \delta) = t,\quad x + \delta \in [0, 1]^n \]
    • C(·) 是分类器输出的类别
    • t 是目标错误类别
    • ‖δ‖_p 是扰动的范数（越小越好）

    4.2 如何处理约束¶

    把约束转化成目标函数的一部分，得到无约束优化问题：

    \[ \min_{\delta} \|\delta\|_p + c · f(x + \delta) \]

    其中 c > 0 是平衡系数，f(·) 是攻击目标函数。

    Carlini & Wagner 提出了 7 种 f(·) 的候选形式，最终推荐的是：

    \[ f(x') = \max\left(\max_{i \neq t} Z(x')_i - Z(x')_t,\; -\kappa\right) \]

    其中 Z(x') 是分类器在 softmax 之前的 logits。κ 控制攻击的置信度（κ 越大，要求错分得越「确信」）。

    直观理解：这个式子的意思是——让正确类别（或非目标类别）的最大 logit 比目标类别的 logit 小至少 κ。当 κ = 0 时，只要目标类 logit 最高就行；κ > 0 时，要求目标类显著领先。

    4.3 Box 约束的三种处理¶

    输入像素必须在 [0, 1] 范围内。C&W 讨论了三种方案：

    方法 怎么做 优缺点
    Projected GD 每步梯度下降后投影回 [0,1] 简单，但传入下一步的不是「真实」值
    Clipped GD 把 x+δ 夹到 [0,1] 再算 loss 梯度信息可能丢失
    Change of Variable 令 δ = ½(tanh(w) + 1) - x，优化 w 自然的 [0,1] 约束，扰动通常更小

    实验结果显示 Projected GD 在 box 约束处理上优于其他方法，但 Change of Variable 找到的扰动通常更小。实际使用时通常用 Projected GD（更稳定）。

    4.4 C&W 和其他方法的本质区别¶

    FGSM / BIM / PGD C&W
    思路 「沿着梯度方向推」 「最小化扰动到刚好错分」
    攻击目标 隐式（增大 loss） 显式（要错分成哪个类）
    扰动大小 由 ε 决定 由优化本身决定（尽量小）
    数学形式 梯度上升 约束优化 → 无约束优化
    攻击强度 强 极强（曾被用于攻破很多防御）

    C&W 攻击被视为白盒攻击的「黄金标准」之一。如果能防御 C&W，基本可以声称对白盒攻击是鲁棒的。但代价是：C&W 计算量远大于 FGSM/PGD，不适合用于对抗训练。

    现在把四篇论文串起来，你会看到一条清晰的演化线：

    演化逻辑：

    1. FGSM：揭示了问题的本质（线性假设），但攻击太弱
    2. BIM：把一步变成多步，同时引入目标攻击，攻击力大幅提升
    3. PGD：BIM 的问题是不够「全局」——加随机初始化 + min-max 框架，让攻击和防御都有了理论基础
    4. C&W：完全换了一条路——不用梯度符号，直接求解最优化问题，aim for 最小扰动

    6. 💭 个人理解¶

    6.1 为什么线性假设这么重要？¶

    Goodfellow 的线性假设是整个对抗攻击领域的「第一性原理」。在我看过的很多后续工作中，这个假设被反复引用但很少被质疑。我的理解是：

    • 它解释了「为什么对抗样本存在」——而不只是「怎么生成」。这比后面所有攻击方法都更根本。
    • 高维空间中，L₁ 范数爆炸这个现象不是神经网络独有的——任何在高维输入上做内积的模型都会遇到。所以 对抗样本不是神经网络的 bug，而是高维学习的 feature。
    • 这个假设的一个自然推论是：对抗鲁棒性可能和模型在权重空间的范数约束有关。事实上后来 Madry 等人的对抗训练本质上就是在约束这个方向。

    6.2 PGD 的随机初始化有多关键？¶

    BIM 到 PGD 的改进看似微小（就加了个随机起点），但背后有一个深刻的道理：确定性攻击容易过拟合到特定的 loss landscape 局部结构，而随机化帮助你探索更广的区域。

    这和 machine unlearning 中的一个核心难题有共鸣——如果你只是确定性地去掉某个数据点的影响，模型可能只忘了「表面的东西」；真正的遗忘需要探索参数空间的不同区域。随机初始化和多步迭代，在思想上是相通的。

    6.3 C&W 为什么对研究 Machine Unlearning 特别相关？¶

    C&W 的目标是「找到最小的扰动让模型出错」。在 machine unlearning 的语境下，这可以直接转译成：

    • 验证遗忘效果：如果一个模型声称「忘记了某个数据」，用 C&W 去攻击它——看它是否仍然对那个数据的对抗版本表现出记忆痕迹。
    • 遗忘后的鲁棒性：遗忘训练通常会改变决策边界，C&W 是测试边界变化的最严格工具之一。

    我正在思考一个问题：遗忘训练后，对抗鲁棒性到底变好还是变差？ 直觉上遗忘可能会「扰乱」决策边界，让模型在某些区域更脆弱；但也可能因为模型容量释放，反而提升了整体鲁棒性。C&W 提供了一个精确的测量工具。

    6.4 这些攻击和 Data Poisoning / Backdoor 的区别¶

    对抗攻击 数据投毒 后门攻击
    攻击时机 推理阶段 训练阶段 训练阶段
    攻击方式 修改单个输入 污染训练数据 植入触发器
    持久性 不持久（每次都要攻击） 持久（影响模型参数） 持久（触发时激活）
    和前者的关系 — 可以看作「训练阶段的对抗攻击」 可以看作「带触发器的投毒」

    这个对比帮我理清了一个一直困惑的问题：对抗攻击和投毒攻击的区别不是「方法不同」，而是「时机不同」。 如果攻击者在训练阶段就有能力污染数据，那他可以用更强的对抗样本来投毒——这就是两者的交叉点。

    7. 后续阅读¶

    • 攻击方法不止这四种。AutoAttack 是目前最常用的标准评测基线，组合了 APGD + APGD-DLR + FAB + Square Attack
    • 黑盒攻击是另一个重要方向——攻击者不知道模型内部，只能查询。方法和白盒完全不同
    • 对抗训练作为防御手段，和这四种攻击方法是「攻防螺旋」的两面
